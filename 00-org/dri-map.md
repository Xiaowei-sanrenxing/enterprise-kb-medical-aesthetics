# DRI 责任地图（极薄组织上下文）

> 不是部门墙,只回答:每条工作流,谁是那个被命名的负责人(DRI)。
> AI Native 公司不靠"部门",靠"每件事一个 DRI"。

| 工作流 / 域 | DRI | 职责 |
|---|---|---|
| 获客引流 | 〔DRI〕 | 内容种草/平台投放/到店钩子，把客流引到店 |
| 咨询面诊转化 | 〔DRI〕 | 在线咨询→到店面诊→成交 |
| 项目交付术后管理 | 〔DRI〕 | 操作SOP/术后跟踪/安全合规 |
| 复购与升单 | 〔DRI〕 | 会员运营/项目升单/老带新 |
| 合规与风控 | 〔DRI〕 | 医疗资质/隐私/纠纷处理（医美特有，权重最高） |
| 知识库治理 | 〔知识库负责人〕 | 准入/定版/权限/同步 |

> 注:初期可单人多角色,团队扩张后每条工作流的 DRI 拆给具体的人,原则不变。
