# 生命周期与定版

> 解决传统文件管理最大的痛:多版本散落,没人知道哪个能拿去交付。

## status 五态（写在每个资产 frontmatter）
| status | 含义 | agent 能否用于交付 |
|---|---|---|
| `草稿` | 在改,未定 | ❌ |
| `定版` | 已确认,生产级 | ✅ 唯一可交付 |
| `待更新` | 定版但已过时 | ⚠️ 提示先更新 |
| `归档` | 旧版,留痕 | ❌ 仅查历史 |
| `废弃` | 作废 | ❌ |

## 铁律
**Agent 永远只用 `status: 定版` 的资产做对外交付。** 看到草稿/待更新先提示用户。

## 版本变更
定版更新 → 旧版改 `归档`,新版标 `定版` + 更新 `updated`。不物理删除,改状态留痕。
