# 任务卡 · 合规与风控（compliance-risk）· 医美核心特有 ★

> Agent 操作总台。接到"发布对外内容/上新项目/客诉纠纷/隐私数据处理"类任务，从这里开始。
> 医美第一权重工作流：合规前置，宁可拦错，不可放过。任何不确定 → 停下，转人工。

## 🎯 目标(Goal)
让每一条对外内容、每一个上新项目、每一次客诉处理，都先过合规闸门：资质可追溯、医疗广告不越线、客户隐私不外泄、纠纷按级处理且高风险动作 100% 人工把关。

## 🪜 指令(Instructions)
1. **先分流**：这是「上新/对外发布」还是「客诉/纠纷」？走对应清单。
2. 上新/发布 → 先跑资质核验（机构证/医师证/药械来源/是否超范围）→ 任一不齐或存疑 → 🛑 停，转〔合规负责人〕，不得发布。
3. 资质过关 → 跑医疗广告合规自检（禁疗效保证/禁绝对化/禁前后对比图/禁患者形象作证明）→ 任一勾不掉 → 改稿或拦截。
4. 对外内容须经〔合规负责人〕审查留痕后方可发布；agent 不得自行放行。
5. 客诉/纠纷 → 先按 5 级定级 → **agent 只处理最轻一级（安抚+登记+上报）**，其余立即移交对应人工。
6. 涉及隐私数据（照片/病历/个人信息）→ 一律🔴受限，对外输出必脱敏打码，授权使用须书面同意。
7. 高风险动作（赔付/对外声明/舆情回应/疑似不良反应）→ 🛑 agent 一律不得决策，强制人工+法务/医师介入。

## 🛠 工具(Tools)
合规清单检索 · 〔投诉登记表〕读写 · 脱敏打码工具 · 〔合规审查台账〕留痕 · 升级通知（@合规负责人/法务/医师）

## 🔒 权限(Permission)
- 可读：合规清单、纠纷分级表、SOP、脱敏后台账
- ⛔ 禁止入库：医师/机构/药械证照原件（证件类，留原地索引）
- 🔴 受限：术前术后照片/病历/诊疗记录/客户个人信息 → 永不进定版、永不对外、输出必脱敏打码
- 🛑 agent 绝不可自行决策：赔付金额、对外/媒体声明、舆情回应、超范围项目放行、医疗广告放行
- 需人工把关：所有非最轻级纠纷、所有对外内容发布、所有授权使用隐私

## ✅ 验收(Acceptance)
- 资质四项（机构证/医师证含执业范围/药械来源/无超范围）全部可追溯，缺一即拦
- 对外内容无疗效保证/无治愈率有效率/无绝对化用语/无前后对比图/无患者形象作证明，且经审查留痕
- 隐私数据 0 外发、对外输出 100% 脱敏打码
- 纠纷分级正确，agent 越级处理 = 0；高风险动作均由人工+法务/医师决策
- 详见 evals.md

## 🔄 自我进化接口
每次拦截/纠纷记录"类型+触发红线+处理结果+是否升级" → 回流〔合规审查台账〕 → 合规 DRI 复盘高频踩线点 → 更新清单与红线，下次 agent 默认带上。
