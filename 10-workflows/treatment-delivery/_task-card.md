# 任务卡 · 项目交付术后管理（treatment-delivery）· 脱敏示范

> Agent 操作总台。接到"术前准入核查/术中记录/术后医嘱/回访跟踪/不良反应响应"类任务，从这里开始。

## 🎯 目标(Goal)
把一个已成交客户，从术前准入安全推进到术后恢复闭环，全程守住两条底线（操作在医师执业范围内、知情同意必签），轻微反应按模板处理、严重并发症立即升级人工+医师。

## 📥 上下文(Context) —— 动手前必读全
1. [`context/项目交付标准流程.md`](context/项目交付标准流程.md) — 术前/术中/术后阶段清单
2. [`context/术后跟踪与不良反应处理.md`](context/术后跟踪与不良反应处理.md) — 回访节奏 + 分级处理
3. [`context/客户主动推送隐私数据处理流.md`](context/客户主动推送隐私数据处理流.md) — 客户微信发来照片/病历的接收·存储·转医师定向通道（🔴禁群转发）
4. 〔知情同意书/医嘱模板〕 — 客户机构真实模板（真实库含此项）
5. 〔不良反应上报流程〕 — 必须遵守的上报路径（真实库含此项）

## 🪜 指令(Instructions)
1. 术前核查四项（资质/项目匹配/禁忌/知情同意），缺一项 → 🔴 停
2. 术前拍照 → 即时归医疗隐私受限区
3. 术中：医师执业范围内操作，三查七对，批号录入可追溯
4. 术后即时医嘱 + 注意事项 + 留观
5. 进回访节奏（24h/3天/7天/1月）分项跟踪
6. 反应分级：🟢轻微按模板 / 🟡需关注通知医师 / 🔴严重立即升级
7. 严重并发症 → 立即人工+医师介入+按规上报，agent 不自行处理

## 🛠 工具(Tools)
〔诊疗记录读写〕 · 〔知情同意/医嘱模板检索〕 · 〔回访任务排程〕 · 〔批号登记表〕

## 🔒 权限(Permission)
- 可读：交付流程、跟踪规范、医嘱/同意书模板
- 🔴 受限：术前术后照片/病历/诊疗记录/客户个人信息 → 永不外发，任何输出必打码
- 须满足：操作在持证医师执业范围内、知情同意书已签、用药用械批号可追溯
- 需人工+医师：🔴 严重不良反应、超执业范围、禁忌命中

## ✅ 验收(Acceptance)
- 术前四项齐 ｜ 知情同意已签 ｜ 操作在执业范围内 ｜ 批号可追溯 ｜ 回访按节奏执行
- 🔴 隐私照片/病历零外发、输出已打码 ｜ 严重反应有升级与上报记录
- 详见 evals.md

## 🔄 自我进化接口
每例交付记录"恢复轨迹 + 是否出现反应 + 卡在哪阶段" → 回流跟踪数据 → 监控 agent 优化回访模板与分级阈值（阈值调整须医师复核）。
