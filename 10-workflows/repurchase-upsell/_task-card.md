# 任务卡 · 复购与升单（repurchase-upsell）· 脱敏示范

> Agent 操作总台。接到"复购提醒/疗程跟进/项目升单/会员转化/老带新转介绍"类任务,从这里开始。

## 🎯 目标(Goal)
让一个已成交客户按项目真实周期完成复购与合理升单,提升 LTV,全程不越合规红线（不诱导过度医疗/不承诺疗效/转介绍物料合规）。

## 📥 上下文(Context) —— 动手前必读全
1. [`context/项目周期复购与升单地图.md`](context/项目周期复购与升单地图.md) — 按项目周期定复购节奏 + 关联升单路径
2. [`context/会员体系与老带新机制.md`](context/会员体系与老带新机制.md) — 会员分层权益 + 转介绍双向激励 + 物料合规
3. 〔客户已做项目档案〕 — 判断复购/升单时机的依据（真实库含此项,🔴 隐私受限）
4. 〔会员权益与价目表〕 — 报价以定版为准（真实库含此项）

## 🪜 指令(Instructions)
1. 读客户已做项目 → 定位所处疗程/维持周期
2. 到复购窗口 → 按周期节奏发提醒（基于真实维持需求,非催单）
3. 判升单时机 → 走升单路径（单项目→疗程包→联合方案→年卡会员）
4. 关联升单只基于真实医学需求 + 客户知情自愿,绝不诱导过度医疗
5. 高价值/稳定客户 → 引导会员升级 → 触发老带新转介绍
6. 转介绍发合规物料（禁前后对比图诱导、禁疗效话术）
7. 全程不承诺疗效,涉医学判断 → 转医师/人工确认

## 🛠 工具(Tools)
〔CRM/会员档案读写〕 · 项目周期提醒排程 · 〔会员权益/价目表检索〕 · 〔转介绍物料库〕

## 🔒 权限(Permission)
- 可读:复购升单地图、会员机制、合规物料、价目表
- 🔴 受限:会员消费数据/客户已做项目/手机微信 → 医疗隐私,永不外发、输出必脱敏
- 需人工确认:超标准会员优惠、关联升单的医学必要性、大额疗程包

## ✅ 验收(Acceptance)
- 复购提醒按项目真实周期 ｜ 升单基于真实需求且客户知情自愿 ｜ 不承诺疗效 ｜ 转介绍物料无前后对比图诱导/无疗效话术 ｜ 会员数据未外发
- 详见 evals.md

## 🔄 自我进化接口
每次复购/升单记录"成功/失败 + 卡在哪个周期节点 + 客户拒绝原因" → 回流会员数据 → 监控 agent 优化复购节奏与升单路径。
