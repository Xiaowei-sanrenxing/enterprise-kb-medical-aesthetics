# 任务卡 · 复购与升单（repurchase-upsell）· 脱敏示范

> Agent 操作总台。接到"复购提醒/疗程跟进/项目升单/会员转化/老带新转介绍"类任务,从这里开始。

## 🎯 目标(Goal)
让一个已成交客户按项目真实周期完成复购与合理升单,提升 LTV,全程不越合规红线（不诱导过度医疗/不承诺疗效/转介绍物料合规）。

## 📥 上下文(Context) —— 动手前必读全
1. [`context/项目周期复购与升单地图.md`](context/项目周期复购与升单地图.md) — 按项目周期定复购节奏 + 关联升单路径
2. [`context/会员体系与老带新机制.md`](context/会员体系与老带新机制.md) — 会员分层权益 + 转介绍双向激励 + 物料合规
3. 〔客户已做项目档案〕 — 判断复购/升单时机的依据（真实库含此项,🔴 隐私受限）
4. 〔会员权益与价目表〕 — 报价以定版为准（真实库含此项）

## 🪜 指令(Instructions)
1. 读客户已做项目 → 定位所处疗程/维持周期
2. 到复购窗口 → 按周期节奏发提醒（基于真实维持需求,非催单）
3. 判升单时机 → 走升单路径（单项目→疗程包→联合方案→年卡会员）
4. 关联升单只基于真实医学需求 + 客户知情自愿,绝不诱导过度医疗
5. 高价值/稳定客户 → 引导会员升级 → 触发老带新转介绍
6. 转介绍发合规物料（禁前后对比图诱导、禁疗效话术）
7. 全程不承诺疗效,涉医学判断 → 转医师/人工确认

## 🛠 工具(Tools)
〔CRM/会员档案读写〕 · 项目周期提醒排程 · 〔会员权益/价目表检索〕 · 〔转介绍物料库〕

## 🔒 权限(Permission)
- 可读:复购升单地图、会员机制、合规物料、价目表
- 🔴 受限:会员消费数据/客户已做项目/手机微信 → 医疗隐私,永不外发、输出必脱敏
- 需人工确认:超标准会员优惠、关联升单的医学必要性、大额疗程包

## ✅ 验收(Acceptance)
- 复购提醒按项目真实周期 ｜ 升单基于真实需求且客户知情自愿 ｜ 不承诺疗效 ｜ 转介绍物料无前后对比图诱导/无疗效话术 ｜ 会员数据未外发
- 详见 evals.md

# 复购与升单 SOP · 人可读总览

> 给人看的全流程。Agent 执行请走 [`_task-card.md`](_task-card.md)。

## 一句话流程
读已做项目定周期 → 复购窗口按节奏提醒 → 判升单时机走升单路径 → 关联升单只基于真实需求 → 高价值客户升会员 → 触发合规老带新。

## 几条铁律
1. **复购按项目真实周期**：光电/注射类有天然疗程与维持周期,按周期提醒,不是催单。
2. **升单不诱导过度医疗**：升单（单项目→疗程包→联合方案→年卡）必须基于真实医学需求 + 客户知情自愿。
3. **关联升单要可解释**：做了 A 推荐 B,须说得清医学逻辑,不确定 → 转医师确认。
4. **绝不承诺疗效**：任何复购/升单话术不得宣称疗效保证、不得绝对化用语。
5. **老带新物料合规**：转介绍禁用前后对比图诱导、禁疗效话术,双向激励用权益不用夸大效果。
6. **隐私红线**：会员消费数据/已做项目/手机微信 🔴 受限,绝不外发。

## 资产清单
- [`context/项目周期复购与升单地图.md`](context/项目周期复购与升单地图.md) — 项目周期 + 复购节奏 + 升单路径
- [`context/会员体系与老带新机制.md`](context/会员体系与老带新机制.md) — 会员分层 + 老带新机制 + 物料合规
- 〔会员权益与价目表〕 — 客户补:报价以定版为准
- 〔转介绍物料库〕 — 客户补:合规审核过的转介绍物料

## 判断题
1. 客户三个月前做了某光电项目,现在该做什么动作?（答：查该项目维持周期,若到复购窗口 → 按周期发复购提醒,基于真实维持需求而非催单）
2. 客户已做注射类基础项目且满意度高,可以推什么?（答：走升单路径——单项目→疗程包→联合方案→年卡会员;关联升单须基于真实医学需求且客户知情自愿）
3. 一个高频到店、消费稳定的老客户,下一步抓什么?（答：引导会员升级,并触发老带新转介绍——给合规推荐礼,双向激励）

## 动作题
4. 判断"做了 A 是否该推荐 B"时,标准依据是什么?（答：真实医学需求 + 客户意愿,逻辑要可解释;不确定 → 转医师确认,绝不为升单凑单）
5. 客户问"做完这个疗程是不是就一定能彻底解决"时,怎么回?（答：绝不承诺疗效/不绝对化,说明个体差异,导向面诊评估,涉医学判断转医师）
6. 设计老带新转介绍物料,第一步审什么?（答：审合规——禁前后对比图诱导、禁疗效话术,激励用权益表达,经合规审核后再用）

## 红线题
7. 为了冲业绩,能不能给客户加推他暂时不需要的高价疗程包?（答：🔴 绝不,升单不得诱导过度医疗,须基于真实需求与客户知情自愿）
8. 转介绍海报里能放老客户的术前术后对比图来吸引新客吗?（答：🔴 绝不,禁前后对比图诱导、禁疗效话术,违反医疗广告合规）
9. 会员消费数据/已做项目记录能写进对外物料或转发吗?（答：🔴 受限,医疗隐私,绝不外发,输出必脱敏打码）

- 全对：可独立上手 ｜ 7-8 对：需带教 ｜ ≤6 对：重读 context（红线题错任一 → 必须重学合规）

# 会员体系与老带新机制（脱敏示范）

> 核心：**用会员分层锁定长期复购,用合规老带新撬动转介绍——激励靠权益,绝不靠夸大效果。**
> （权益/礼品为示意,客户填自己机构的真实设置）

## 一、会员分层（示意）

- **〔储值会员〕**：储值享〔权益占位〕,适合高频护理客户
- **〔疗程会员〕**：锁定疗程包,享〔疗程价/优先排期〕
- **〔年卡会员〕**：长期绑定,享〔年度权益/专属顾问〕

> 权益设计原则：用确定性权益（价格/排期/服务）吸引,**不得以疗效承诺作为权益卖点**。

## 二、复购激励
- 到维持周期的会员 → 专属复购权益（衔接 [[项目周期复购与升单地图]]）
- 升级会员 = 升单路径终点,稳定客户优先引导

## 三、老带新转介绍机制
- **推荐礼设计**：〔推荐礼占位〕,双向激励——老客户得〔权益〕,新客户得〔权益〕
- **触发时机**：满意度高、消费稳定的老客户
- **物料合规（红线）**：
  - 🔴 禁用前后对比图诱导
  - 🔴 禁疗效话术、禁绝对化用语
  - 物料须经合规审核后再用,激励用权益表达

## 四、数据隐私（红线）
- 🔴 会员消费数据 / 已做项目记录 / 手机微信 = 医疗隐私,受限,绝不外发、绝不进对外物料,输出必脱敏。

## 用法
高价值客户引导会员升级 → 满意稳定后触发老带新 → 发合规物料。会员复购节奏见 [[项目周期复购与升单地图]]；新客承接交给 [[在线咨询到到店转化SOP]]。

# 项目周期复购与升单地图（脱敏示范）

> 核心：**医美项目天然有疗程周期与维持周期,按真实周期设计复购节奏——是医学维持,不是催单。**
> （周期/项目为示意,客户填自己机构的真实方案,涉医学判断须医师确认）

## 一、按项目周期定复购节奏

| 项目类型（示意） | 疗程周期 | 维持周期 | 复购提醒窗口 |
|---|---|---|---|
| 〔光电类〕 | 多次为一疗程,间隔〔示意 3-4 周〕 | 〔示意 半年-1 年〕 | 单次后到下次间隔前、维持期临近 |
| 〔注射类〕 | 单次见效 | 〔示意 4-6 个月〕代谢后 | 维持期临近 |
| 〔皮肤管理/护理类〕 | 周期性护理 | 按护理周期 | 每个护理周期到点 |

> 提醒话术基于"真实维持需求",绝不承诺疗效、不绝对化。到点提醒 ≠ 催单。

## 二、升单路径（由浅入深）

```
单项目  →  疗程包  →  联合方案  →  年卡/会员
（先体验）  （锁周期）  （多项目协同）  （长期绑定 → 见 [[会员体系与老带新机制]]）
```

## 三、关联升单（做了 A 通常需要 B）

| 已做项目 | 可关联（须真实医学逻辑） | 红线 |
|---|---|---|
| 〔做了 A〕 | 〔通常配合 B 巩固/协同〕 | 须客户知情自愿 |
| …… | …… | 不得诱导过度医疗 |

> ⚠️ **关联升单铁律**：只基于真实医学需求 + 客户意愿,逻辑要可解释;不确定 → 转医师确认,绝不为凑单加推不需要的项目。

## 用法
先读客户已做项目定位周期 → 到窗口按节奏提醒 → 判升单时机走路径。高价值客户导向 [[会员体系与老带新机制]]；涉术后维持衔接 [[项目交付标准流程（术前-术中-术后）]]。

# 权限分级 · 四级

> 企业库和个人库最硬的区别。没有权限治理 = AI 资料垃圾场 + 隐私事故。

| 级别 | 含义 | 典型资产 | agent 行为 |
|---|---|---|---|
| 🟢 公开 | 可对外发布 | 对外物料、方法论、介绍 | 可写入对外产物 |
| 🟡 内部 | 团队内用 | 话术、SOP、运营数据 | 内部可用,不对外 |
| 🔴 受限 | 隐私/敏感 | 客户对话、通话记录、手机号、名单 | **绝不写入对外产物,只内部分析** |
| ⛔ 禁止入库 | 不该进库 | 证件、银行信息、未授权第三方资料 | 拒绝入库 |

## agent 红线
1. 🔴 受限资产 → 输出时必须脱敏/打码,绝不原文外发

# 医美行业 · 合规与风控默认值（行业特有，权重最高）

> 合成进成品库 `90-governance/行业合规默认.md`。医美默认更严，客户只可收紧不可放松。

## 默认权限分级（医美收紧版）
| 资产类型 | 默认级别 |
|---|---|
| 客户病历/术前术后照片/诊疗记录 | 🔴 受限（医疗隐私，永不对外、输出必脱敏打码）|
| 客户个人信息（姓名/手机/微信）| 🔴 受限 |
| 医师资质/机构许可/药械证照 | ⛔ 禁止入库（证件类，留原地索引）|
| 项目话术/咨询SOP/运营数据 | 🟡 内部 |
| 对外科普/品牌介绍 | 🟢 公开（须经合规审核，禁违反医疗广告法）|

## 医美红线（agent 必须遵守）
1. **医疗广告合规**：对外内容不得宣称疗效保证、不得用前后对比图诱导、不得绝对化用语。
2. **诊疗隐私**：术前术后照片/病历属医疗隐私，🔴 受限，绝不进定版、绝不对外、输出必打码。
3. **资质追溯**：涉及项目须可追溯医师资质与药械合规，不确定→人工确认。
4. **纠纷处理**：客诉/纠纷走 `compliance-risk` 工作流，高风险动作（赔付/对外声明）必须人工把关。

> 合规与风控是医美独立工作流 `compliance-risk`，不是附属——这是医美区别于其他行业的核心。
