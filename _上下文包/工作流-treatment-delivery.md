# 项目交付术后管理 SOP · 人可读总览

> 给人看的全流程。Agent 执行请走 [`_task-card.md`](_task-card.md)。

## 一句话流程
术前核查四项（缺一就停）→ 拍照归隐私区 → 医师执业范围内操作+批号可追溯 → 术后即时医嘱+留观 → 按 24h/3天/7天/1月回访 → 反应分级处理，严重并发症立即升级人工+医师。

## 几条铁律
1. **两条准入底线**：操作必须在持证医师执业范围内、术前知情同意书必签——缺一不开做。
2. **项目匹配先于操作**：客户诉求与项目适应证不符，由医师面诊决定，不硬做。
3. **批号可追溯**：用药用械批号逐项登记，无追溯不操作。
4. **隐私红线**：术前术后照片/病历/诊疗记录 🔴 受限，永不外发，输出必打码。
5. **严重反应不自判**：感染/栓塞等严重并发症 = 🔴 立即人工+医师介入+按规上报，agent 绝不自行处理。

## 详细资产
- [`context/项目交付标准流程.md`](context/项目交付标准流程.md) — 术前/术中/术后阶段清单
- [`context/术后跟踪与不良反应处理.md`](context/术后跟踪与不良反应处理.md) — 回访节奏 + 分级处理

# 客户主动推送隐私数据处理流（操作 SOP）

> 补隐私铁律的盲区：铁律讲"机构内部产生→存受限区→对外脱敏"，本流程专管**客户从微信等外部主动发来病历/照片**这一最高频入口。
> 核心：客户主动发来 ≠ 可以随便处理。一旦进入机构，立即按 🔴 受限医疗隐私管。

## 标准动作流（agent 收到客户主动推送的照片/病历时）
1. **即时定性**：客户发来的术后照/病历/诊疗记录 = 🔴 受限医疗隐私，从落地那一刻起按受限管。
2. **不在线诊断**：agent 绝不判断"正常/不正常"（医学判断须医师），先安抚 + 收集客观信息（部位/时间/症状）。
3. **急症先行**：若描述含 🔴 表现（剧痛/发白/花斑/化脓/视力异常等）→ 立即走 [[术后跟踪与不良反应处理]] 急症升级，不等流转。
4. **合规入库**：照片/记录存入该客户 `_受限/` 档案专属分区，**不落入草稿区、不进任何对外/共享目录**。
5. **转医师走定向通道**：需医师看 → 走**一对一定向通道**（指定医师私发/专用病历系统），**🔴 严禁丢进多人医师群/工作群/对外群**（群转发=超授权扩散，最高发隐私事故）。转发时只给必要的医师，附最小必要信息（最小授权原则）。
6. **留痕**：登记"收到时间/客户/处理人/转给谁"，便于追溯。

## 红线（agent 绝不做）
- 🔴 把客户照片/病历转发到任何群（含内部医师群/工作群）
- 🔴 在线上对诊疗结果下判断
- 🔴 把受限照片用于案例/对外（即使脱敏也须另走授权流程）

## 不确定时
默认按受限处理 + 转 DRI/医师，宁可保守不可扩散。

# 术后跟踪与不良反应处理

> 核心：**轻微反应可按模板安抚+提醒，严重并发症 = 🔴 必须人工+医师介入并按规上报，agent 不得自行判断处理。**
> （回访话术/恢复期模板客户填自己机构的真实版本）

## 回访节奏（按项目分项跟踪）
| 时间点 | 重点 | 动作 |
|---|---|---|
| 术后 24h | 即时反应、红肿渗血 | 主动回访，确认无急性异常 |
| 术后 3 天 | 恢复期高峰、肿胀淤青 | 按〔恢复期模板〕提醒护理 |
| 术后 7 天 | 拆线/消肿/初步效果 | 确认恢复轨迹，约复诊 |
| 术后 1 月 | 最终效果、满意度 | 评估、记录、按需安排复查 |

## 不良反应分级与处理
| 分级 | 表现（示意） | 处理 |
|---|---|---|
| 🟢 轻微 | 轻度红肿、淤青、短暂胀痛 | 按〔恢复期模板〕安抚+护理提醒，记录观察 |
| 🟡 需关注 | 持续红肿、硬结、低热 | 提醒就诊复查，通知医师评估，记录 |
| 🔴 严重 | 感染、剧痛、皮肤发白/花斑（疑栓塞）、视力异常、过敏休克 | **立即指导就医/转诊 → 人工+医师介入 → 按规上报**，agent 不得自行判断处理 |

## 何时必须立即就医/转诊
- 出现 🔴 任一表现 → 第一时间指导客户立即就医，同步医师，不得拖延。
- 疑似血管栓塞（皮肤发白/剧痛/花斑）属急症 → 争分夺秒，立即人工介入。

## 红线
- 严重不良反应 = 🔴 必须人工 + 医师介入 + 按不良反应上报流程上报，**agent 绝不自行诊断或处理**。
- 术后照片/病历 = 🔴 医疗隐私，受限永不外发，输出必打码。

# 项目交付标准流程（术前-术中-术后）

> 核心：**操作必须在持证医师执业范围内、术前知情同意书必签——两条缺一不可，否则不得开始。**
> （以下为标准动作清单，客户填自己机构的真实模板与项目名）

## 术前 · 准入核查（缺一项就停）
- [ ] 核对操作者资质：项目须落在**持证医师执业范围内**，超范围 → 🔴 停止、转诊
- [ ] 核对项目匹配：客户诉求 ↔ 〔项目名〕适应证一致；**不匹配不开做**，由医师面诊决定
- [ ] 过敏史 + 禁忌症核查：逐项问诊填表，命中禁忌 → 🔴 停止
- [ ] 签署〔知情同意书〕：风险/恢复期/费用讲清，**客户本人签字**，未签不得开始
- [ ] 术前拍照存档：照片即时归入 🔴 医疗隐私受限区，永不外发、输出必打码

## 术中 · 操作执行
- [ ] 仅由持证医师在执业范围内操作，agent/助理不得替代医疗判断
- [ ] 三查七对意识：查客户/查项目/查部位，对身份对药械
- [ ] 记录用药用械**批号可追溯**：〔药械批号登记表〕逐项录入

# 权限分级 · 四级

> 企业库和个人库最硬的区别。没有权限治理 = AI 资料垃圾场 + 隐私事故。

| 级别 | 含义 | 典型资产 | agent 行为 |
|---|---|---|---|
| 🟢 公开 | 可对外发布 | 对外物料、方法论、介绍 | 可写入对外产物 |
| 🟡 内部 | 团队内用 | 话术、SOP、运营数据 | 内部可用,不对外 |
| 🔴 受限 | 隐私/敏感 | 客户对话、通话记录、手机号、名单 | **绝不写入对外产物,只内部分析** |
| ⛔ 禁止入库 | 不该进库 | 证件、银行信息、未授权第三方资料 | 拒绝入库 |

## agent 红线
1. 🔴 受限资产 → 输出时必须脱敏/打码,绝不原文外发
2. 超标准的报价/优惠/对外承诺 → 必须人工确认
3. 不确定权限 → 默认按更高一级处理,问用户


## 治理 · 行业合规默认.md

# 医美行业 · 合规与风控默认值（行业特有，权重最高）

> 合成进成品库 `90-governance/行业合规默认.md`。医美默认更严，客户只可收紧不可放松。

## 默认权限分级（医美收紧版）
| 资产类型 | 默认级别 |
|---|---|
| 客户病历/术前术后照片/诊疗记录 | 🔴 受限（医疗隐私，永不对外、输出必脱敏打码）|
| 客户个人信息（姓名/手机/微信）| 🔴 受限 |
| 医师资质/机构许可/药械证照 | ⛔ 禁止入库（证件类，留原地索引）|
| 项目话术/咨询SOP/运营数据 | 🟡 内部 |
| 对外科普/品牌介绍 | 🟢 公开（须经合规审核，禁违反医疗广告法）|

## 医美红线（agent 必须遵守）
1. **医疗广告合规**：对外内容不得宣称疗效保证、不得用前后对比图诱导、不得绝对化用语。
2. **诊疗隐私**：术前术后照片/病历属医疗隐私，🔴 受限，绝不进定版、绝不对外、输出必打码。
3. **资质追溯**：涉及项目须可追溯医师资质与药械合规，不确定→人工确认。
4. **纠纷处理**：客诉/纠纷走 `compliance-risk` 工作流，高风险动作（赔付/对外声明）必须人工把关。

> 合规与风控是医美独立工作流 `compliance-risk`，不是附属——这是医美区别于其他行业的核心。
