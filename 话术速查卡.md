# 🗣 〔机构名〕 知识库 · 傻瓜话术速查卡（医美版）

> 不知道怎么跟 AI 开口？**照着念，把 `<尖括号>` 里的换成你的真实内容就行。**
> 适用于任何 AI：Claude Code / 飞书 AI / 网页版 ChatGPT、Claude 都能用。
> 用 Claude Code 的，还能用「斜杠版」——敲个斜杠点名，连话都不用编。

---

## 🧭 一、不知道这事归哪条流程？——先让 AI 带路

> **「读 AGENTS.md，告诉我『<我要做的事>』该走哪条工作流，然后带上资料帮我做。」**

这是**万能开头**。不确定时永远先说这句，AI 自己会找对路。

本库目前有这几条工作流可选：

- **获客引流**（`traffic-acquisition`）— 内容种草/平台投放/到店钩子，把客流引到店
- **咨询面诊转化**（`consult-conversion`）— 在线咨询→到店面诊→成交
- **项目交付术后管理**（`treatment-delivery`）— 操作SOP/术后跟踪/安全合规
- **复购与升单**（`repurchase-upsell`）— 会员运营/项目升单/老带新
- **合规与风控**（`compliance-risk`）— 医疗资质/隐私/纠纷处理（医美特有，权重最高）

---

## ✍️ 二、用库里的资料做创作 / 方案 / 干活（最高频）

> **「用 `<工作流名>` 工作流 context 里的资料，帮我 `<写一篇 / 做一版 / 整理一份>` `<主题>`。」**

例（套本行业真实工作流）：
- `用 traffic-acquisition 工作流 context 里的资料，帮我写一篇关于「获客引流」的内容。`
- `用 consult-conversion 工作流的资料，帮我做一版「咨询面诊转化」的方案。`

> 好处：AI 带着你**真实的方法和数据**写，不是凭空编，还自动避开红线（比如不外发客户隐私）。

---

## 📥 三、有新资料，想存进知识库（喂料 / 回流）

**小文本**（话术 / SOP / 模板 / 复盘）→ 想进库：
> **「把这份 `<资料名>` 按 `intake-rules` 入库，归到对应工作流。」**

AI 会自动：判断归哪条流 → 放进 `context/` → 补好元数据 → 先存草稿等你确认。

**斜杠版（装了 Claude Code）**：输入 `/company-brain-ingest`，把文件一丢，全自动。

---

## 📎 四、大文件（PPT / 视频 / 录音 / 证照 / 合同）

> **大文件不进库！** 只登记一行索引：
> **「把这个文件登记到 `30-raw/原始素材索引.md`，关联 `<工作流名>`，别搬进库。」**

例：`把这份"<文件名>"登记到 30-raw，关联 traffic-acquisition。`

---

## 🩺 五、给知识库做体检（用久了清理一下）

> **「对照各工作流 `evals.md`，给这个库做一次体检，列出缺口和问题。」**

**斜杠版（装了 Claude Code）**：输入 `/company-brain-audit`，自动扫 + 出报告。

---

## 🔒 六、把草稿转正式（定版）/ 同步给团队

转正式：
> **「这份 `<资料名>` 我确认没问题了，把 status 改成『定版』。」**

同步团队（如用飞书）：
> **「把所有『定版』且非受限的资产，按 `90-governance/feishu-sync.md` 同步到飞书。」**

---

## 🚦 七、三条红线（嘴上也得记住）

1. **客户隐私**（真名 / 手机 / 微信 / 配方 / 进货价等命脉）→ 标 `受限`，**永不对外、永不进定版**。
2. **别在母版上直接干某一单的活** → 具体业务在自己的项目文件夹干，精华才回流。
3. **大文件不搬进库** → 只在 `30-raw` 挂索引。

---

> 💡 记不住没关系：**90% 的情况，第一句和第二句就够用了。**
> 想不起来怎么说，就念第一句「读 AGENTS.md，这事该走哪条工作流」，AI 会接着引导你。
